Módulo 1: Exercício de Lógica

Pratique os conceitos de algoritmos e lógica com desafios do cotidiano.

Introdução

Este módulo foi criado para fortalecer seu raciocínio lógico através de desafios práticos. Cada exercício propõe um problema do cotidiano e, ao clicar no botão, você poderá visualizar a resposta e a explicação detalhada.

Exercício 1: Desafio da Sequência Numérica

Dada a sequência: 2, 4, 8, 16, ... Qual é o próximo número? Explique sua lógica.

Resposta: 32

Explicação: Cada número é o dobro do anterior. Assim, 16 x 2 = 32.

Exercício 2: Ordenação de Nomes

Organize a lista de nomes em ordem alfabética: ["Mariana", "Ana", "Carlos", "Bruno"]. Escreva um pseudocódigo para resolver o problema. 

Início
   Lista ← ["Mariana", "Ana", "Carlos", "Bruno"]
   Enquanto a lista não estiver ordenada:
       Para cada par de nomes consecutivos:
           Se o primeiro for maior que o segundo, troque-os
       FimPara
   FimEnquanto
   Exiba Lista
Fim

Explicação: Este pseudocódigo descreve um algoritmo de ordenação, similar ao Bubble Sort, onde os elementos são trocados de posição até que a lista esteja em ordem alfabética.

Exercício 3: Cálculo da Média

Dada a lista de números: [10, 20, 30, 40, 50]. Calcule a média. 

Início
   Lista ← [10, 20, 30, 40, 50]
   Soma ← 10 + 20 + 30 + 40 + 50 = 150
   Média ← 150 / 5 = 30
   Exiba Média
Fim

Explicação: O algoritmo soma todos os elementos e divide pela quantidade de itens (5) para calcular a média.

Exercício 4: Verificação de Paridade

Crie um pseudocódigo que determine se um número é par ou ímpar. 

Início
   Leia Número
   Se (Número mod 2 == 0) então
      Escreva "Número par"
   Senão
      Escreva "Número ímpar"
   FimSe
Fim

Explicação: Utilizamos o operador módulo para obter o resto da divisão por 2. Se o resto for zero, o número é par; caso contrário, é ímpar.

Exercício 5: Cálculo do Fatorial

Escreva um pseudocódigo para calcular o fatorial de um número utilizando recursividade. 

Função fatorial(n)
   Se n <= 1 então
      Retorne 1
   Senão
      Retorne n * fatorial(n - 1)
FimFunção

Explicação: A função chama a si mesma com o valor de n diminuído de 1 até atingir a condição base (n <= 1), retornando 1 e calculando o fatorial.

Exercício 6: Verificação de Palíndromo

Crie um pseudocódigo para verificar se uma palavra é um palíndromo (lida da mesma forma de frente para trás e de trás para frente). 

Início
   Leia Palavra
   PalavraInvertida ← inverter(Palavra)
   Se Palavra == PalavraInvertida então
      Escreva "É um palíndromo"
   Senão
      Escreva "Não é um palíndromo"
   FimSe
Fim

Explicação: O algoritmo inverte a palavra e compara com a original. Se forem iguais, é um palíndromo.

Exercício 7: Soma de Dígitos

Dado um número, some todos os dígitos que o compõem. Por exemplo, para 123, a soma é 1 + 2 + 3 = 6. 

Início
   Leia Número
   Inicialize Soma ← 0
   Enquanto Número > 0 faça
       Dígito ← Número mod 10
       Soma ← Soma + Dígito
       Número ← Número / 10 (descarte os decimais)
   FimEnquanto
   Exiba Soma
Fim

Explicação: O algoritmo extrai cada dígito usando o operador módulo e acumula a soma. A divisão por 10 elimina o dígito já somado.

Exercício 8: Inversão de String

Dada uma palavra, inverta sua ordem. Por exemplo, "casa" se torna "asac". 

Início
   Leia Palavra
   Inicialize PalavraInvertida ← ""
   Para i de comprimento(Palavra) até 1 faça
       PalavraInvertida ← PalavraInvertida + Palavra[i]
   FimPara
   Exiba PalavraInvertida
Fim

Explicação: O algoritmo percorre a palavra do final para o início e constrói uma nova string com os caracteres na ordem inversa.

Dicas Gerais para Praticar a Lógica

  • Escreva o pseudocódigo antes de codificar.
  • Utilize fluxogramas para visualizar o fluxo dos algoritmos.
  • Pratique resolvendo desafios cotidianos, como organizar tarefas ou manipular números.
  • Discuta suas soluções com colegas para aprimorar o raciocínio.

Conclusão

Praticar exercícios de lógica é fundamental para desenvolver a habilidade de decompor problemas e criar algoritmos eficientes. Ao explorar desafios variados, você fortalece sua base para a programação e se prepara para enfrentar problemas mais complexos.
