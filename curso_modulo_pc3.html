Curso Módulo 3: Conceitos Básicos de Programação

Aprenda os fundamentos essenciais da programação e construa uma base sólida para desenvolver suas aplicações.

Conceitos Fundamentais do Programador

Antes de mergulhar nos detalhes do código, é importante conhecer alguns conceitos essenciais que formam a base do desenvolvimento de software:

  • Backend: Refere-se à parte do sistema que roda no servidor. É responsável pelo processamento dos dados, lógica de negócios e comunicação com bancos de dados. O backend funciona "por trás dos panos" e é fundamental para que a aplicação funcione corretamente.
  • Front-end: É a parte do sistema com a qual o usuário interage, como a interface gráfica de um site ou aplicativo. Tudo o que você vê – botões, formulários, menus – faz parte do front-end.
  • Fullstack: Um desenvolvedor fullstack possui conhecimentos tanto de front-end quanto de backend, conseguindo criar aplicações completas e integradas.
  • Endpoints e APIs: Uma API (Interface de Programação de Aplicações) define regras para que diferentes softwares se comuniquem. Um endpoint é o ponto final de uma API – uma URL onde um serviço pode ser acessado. Por exemplo, ao acessar um site de clima, seu aplicativo faz uma requisição a um endpoint que retorna os dados meteorológicos.

1. O que é Programação?

Programação é o processo de criar um conjunto de instruções que dizem a um computador como executar uma tarefa. É como escrever uma receita detalhada, onde cada passo é fundamental para o resultado final.

Para criar códigos, você precisa entender o problema, planejar a solução (usando fluxogramas ou pseudocódigo) e transcrever essa lógica para uma linguagem de programação.

2. Como se Cria Códigos?

Criar um código envolve as seguintes etapas:

  1. Entender o problema: Analise e divida a tarefa em etapas menores.
  2. Planejar o algoritmo: Descreva a solução utilizando linguagem natural, fluxogramas e pseudocódigo.
  3. Escolher uma linguagem: Selecione a linguagem que melhor se adequa ao projeto.
  4. Escrever o código: Transcreva o algoritmo para a sintaxe da linguagem escolhida, definindo variáveis, funções e estruturas de controle.
  5. Testar e depurar: Execute o código, identifique e corrija erros usando ferramentas de debugging.
  6. Documentar: Comente o código para facilitar a manutenção e a colaboração futura.

3. O que é uma IDE?

Uma IDE (Ambiente de Desenvolvimento Integrado) é um software que reúne diversas ferramentas essenciais para o desenvolvimento, como editor de código, compilador/interpreter, depurador e gerenciador de projetos. Ela torna o processo de programar mais eficiente e organizado.

Exemplos de IDEs incluem o Visual Studio Code, Eclipse, IntelliJ IDEA e PyCharm.

4. Como Funciona um Programa?

Um programa funciona através da execução sequencial de instruções, divididas basicamente em três partes:

  • Entrada: Dados recebidos do usuário ou de sistemas externos.
  • Processamento: Aplicação da lógica para transformar os dados.
  • Saída: Exibição ou armazenamento do resultado.

Por exemplo, um programa de calculadora recebe números, processa uma operação e retorna o resultado.

5. Vertentes no Mundo da Programação

A programação é uma área vasta, com diversas especializações:

  • Desenvolvimento Web: Criação de sites e aplicações web, tanto no front-end quanto no back-end.
  • Desenvolvimento Mobile: Criação de aplicativos para smartphones e tablets.
  • Desenvolvimento de Jogos: Criação de jogos digitais.
  • Data Science e Machine Learning: Análise de dados e criação de modelos preditivos.
  • Desenvolvimento de Sistemas: Criação de softwares corporativos e robustos.
  • Programação de Baixo Nível: Trabalho próximo ao hardware, como em linguagens de montagem e C.

6. Endpoints e APIs – Uma Analogia Simples

APIs (Interfaces de Programação de Aplicações) são conjuntos de regras que permitem a comunicação entre diferentes softwares. Um endpoint é o endereço (URL) onde um serviço está disponível.

Uma analogia simples: imagine um restaurante. O cardápio (endpoint) lista os pratos disponíveis. O garçom (API) recebe seu pedido e leva-o à cozinha (backend), retornando com o prato pronto para ser servido no balcão (front-end). Desenvolvedores que trabalham com fullstack cuidam tanto da parte que o usuário vê quanto da lógica interna do sistema.

7. Estrutura de um Programa

Um programa geralmente é dividido em três partes:

  • Entrada: Dados que o programa recebe.
  • Processamento: A lógica que transforma os dados.
  • Saída: O resultado final, seja exibido na tela, salvo ou enviado a outro sistema.

Exemplo: Um programa de calculadora que recebe dois números, soma-os e exibe o resultado.

8. Sintaxe e Semântica

Sintaxe é o conjunto de regras para escrever o código corretamente – semelhante à gramática em uma língua natural. Se a sintaxe estiver errada, o programa não será executado.

Semântica refere-se ao significado dos comandos. Mesmo que o código esteja sintaticamente correto, se a lógica (semântica) estiver equivocada, o resultado não será o esperado.

9. Variáveis e Tipos de Dados

Variáveis são espaços na memória que armazenam valores, enquanto os tipos de dados definem a natureza desses valores (números, textos, booleanos, etc.).

  • Exemplo em JavaScript: 
    let nome = "Maria";
const idade = 30;
  • Exemplo em Python: 
    nome = "Maria"
idade = 30
  • Exemplo em Java: 
    String nome = "Maria";
int idade = 30;
boolean ativo = true;

10. Estruturas de Controle

Estruturas de controle são usadas para direcionar o fluxo de execução do programa, permitindo decisões e repetições.

Exemplo de condicional (if-else) em JavaScript: 

if (idade >= 18) {
  console.log("Maior de idade");
} else {
  console.log("Menor de idade");
}

Exemplo de laço (for) em Python: 

for i in range(1, 6):
    print("Número:", i)

11. Funções e Procedimentos

Funções são blocos de código reutilizáveis que executam tarefas específicas. Elas ajudam a modularizar o programa e tornar o código mais organizado e fácil de manter.

Exemplo de função em JavaScript: 

function saudacao(nome) {
  return "Olá, " + nome + "!";
}
console.log(saudacao("Carlos"));

Exemplo de função em Python: 

def saudacao(nome):
    return "Olá, " + nome + "!"
print(saudacao("Carlos"))

12. Paradigmas de Programação

Paradigmas de programação são estilos e abordagens para organizar e estruturar o código. Os principais paradigmas incluem:

  • Imperativo: Baseado na execução sequencial de comandos que modificam o estado do sistema.
  • Estruturado: Organiza o código em blocos e funções para melhorar a clareza e manutenção.
  • Orientado a Objetos (POO): Estrutura o código em objetos que representam entidades do mundo real, promovendo reutilização e modularidade.
  • Funcional: Foca em funções puras, imutabilidade e evita efeitos colaterais, facilitando a previsão do comportamento do código.

13. Exemplo Prático Integrado

Este exemplo combina vários conceitos básicos: o programa solicita o nome e a idade do usuário, verifica se ele é maior de idade e exibe uma mensagem personalizada.

Exemplo em JavaScript:

// Função para verificar a idade e retornar uma mensagem
function verificarIdade(nome, idade) {
  if (idade >= 18) {
    return "Olá, " + nome + "! Você é maior de idade.";
  } else {
    return "Olá, " + nome + "! Você é menor de idade.";
  }
}
let usuarioNome = prompt("Digite seu nome:");
let usuarioIdade = parseInt(prompt("Digite sua idade:"));
alert(verificarIdade(usuarioNome, usuarioIdade));

Explicação: A função verificarIdade utiliza uma estrutura condicional if-else para determinar se o usuário é maior de idade. As funções prompt e alert interagem com o usuário, capturando dados e exibindo o resultado.

Exemplo em Python:

def verificar_idade(nome, idade):
    if idade >= 18:
        return "Olá, " + nome + "! Você é maior de idade."
    else:
        return "Olá, " + nome + "! Você é menor de idade."

usuario_nome = input("Digite seu nome: ")
usuario_idade = int(input("Digite sua idade: "))
print(verificar_idade(usuario_nome, usuario_idade))

14. Outros Conceitos Básicos

  • Controle de Versão: Ferramentas como Git ajudam a acompanhar e gerenciar alterações no código.
  • Compilação vs. Interpretação: Alguns programas são compilados em código de máquina antes da execução, enquanto outros são interpretados linha a linha.
  • Debugging: Processo de encontrar e corrigir erros no código, utilizando depuradores e logs.
  • Documentação: A prática de comentar o código e criar manuais para facilitar a manutenção.
