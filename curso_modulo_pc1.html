Curso Módulo 1: Introdução aos Algoritmos e Lógica

Construa uma base sólida para o pensamento computacional com conceitos, exemplos e representações detalhadas.

1. O que é um Algoritmo?

Um algoritmo é um conjunto finito, ordenado e preciso de instruções que, quando executadas, solucionam um problema ou realizam uma tarefa. Ele pode ser comparado a uma receita, onde cada passo deve ser seguido na ordem correta para se obter o resultado esperado.

Exemplo prático – Trocar uma lâmpada:

  1. Verificar se a lâmpada está queimada;
  2. Desligar o interruptor;
  3. Retirar a lâmpada antiga;
  4. Instalar a nova lâmpada;
  5. Ligar o interruptor para testar o funcionamento.

Este exemplo simples mostra como um algoritmo pode ser aplicado até mesmo em tarefas cotidianas.

2. Representações Algorítmicas

Para facilitar o entendimento e a implementação, os algoritmos podem ser representados de várias formas:

  • Linguagem Natural: Utiliza a descrição em palavras comuns para explicar os passos. Exemplo: "Verifique se a lâmpada está queimada. Se estiver, desligue o interruptor e troque a lâmpada."
  • Fluxograma: Um diagrama que representa o fluxo de execução do algoritmo com símbolos gráficos para processos, decisões e conexões.
  • Pseudocódigo: Uma maneira intermediária que utiliza estruturas similares às linguagens de programação, mas sem a necessidade de seguir uma sintaxe rígida.

  • Linguagem de Programação: A implementação real do algoritmo em código, utilizando linguagens como Python, JavaScript, entre outras.

3. Linguagem de Programação

As linguagens de programação permitem que os algoritmos sejam traduzidos em comandos que o computador executa. Cada linguagem tem sua sintaxe e particularidades, mas todas cumprem o papel de transformar a lógica do algoritmo em instruções práticas.

Exemplos: Python, Java, C, JavaScript, C++.

4. Paradigmas de Programação

Paradigmas de programação são abordagens que definem como o código é estruturado e como os problemas são resolvidos. Eles influenciam a maneira de pensar e de organizar o desenvolvimento do software. Entre os principais paradigmas, podemos destacar:

  • Paradigma Imperativo: Consiste em uma sequência de comandos que modificam o estado do sistema.
  • Paradigma Estruturado: Organiza o código em blocos bem definidos (como funções, loops e condicionais) para facilitar a manutenção e a leitura.
  • Paradigma Orientado a Objetos (POO): Estrutura o código em objetos, que são instâncias de classes que representam entidades do mundo real. Isso favorece a modularidade e a reutilização.
  • Paradigma Declarativo: Foca no “o que” deve ser feito, deixando de lado os detalhes de como executar a tarefa, comum em linguagens funcionais e em consultas de bancos de dados.

5. Exemplos de Linguagens de Programação

Cada linguagem pode ser melhor utilizada em um ou mais paradigmas. Veja alguns exemplos:

  • Python: Versátil – suporta programação imperativa, estruturada e orientada a objetos.
  • Java: Forte na orientação a objetos.
  • C: Enfatiza a programação imperativa e estruturada.
  • JavaScript: Amplamente usado tanto para programação imperativa quanto para orientada a objetos.
  • C++: Combina recursos da programação orientada a objetos com a procedural.

6. Tipos de Dados

Os tipos de dados determinam o tipo de valor que uma variável pode armazenar e como esses valores podem ser manipulados. Entre os tipos primitivos, destacam-se:

  • Números: Podem ser inteiros (ex.: 10, -5) ou reais (ex.: 3.14, -0.001).
  • Letras (Strings): Conjunto de caracteres formando palavras ou frases (ex.: "Olá, mundo!").
  • Binários: Valores em base 2 (0 e 1), fundamentais para operações a nível de hardware e para a representação digital de informações.

7. O que é uma Variável?

Uma variável é um espaço na memória do computador que guarda um valor que pode ser alterado durante a execução do programa. Ela permite que os dados sejam armazenados, recuperados e manipulados conforme necessário.

8. Como Declarar uma Variável?

A forma de declarar uma variável varia conforme a linguagem utilizada. Em linguagens dinâmicas, a declaração é simples e não requer especificação de tipo; em linguagens tipadas, é necessário indicar o tipo do dado.

Exemplo em JavaScript (não tipada):

let nome = "João";
const idade = 25;

Exemplo em Python (não tipada):

nome = "João"
idade = 25

Exemplo em Java (tipada):

String nome = "João";
int idade = 25;

9. Linguagem Tipada x Não Tipada

Em linguagens tipadas, como Java ou C, o tipo de cada variável deve ser declarado explicitamente. Já em linguagens não tipadas ou de tipagem dinâmica, como Python ou JavaScript, o tipo é inferido automaticamente.

10. Termos Frequentemente Utilizados

Durante o estudo de algoritmos e programação, alguns termos são recorrentes:

  • Programa: Conjunto de instruções que executa uma tarefa específica.
  • Código: O texto escrito em uma linguagem de programação que implementa o algoritmo.
  • Back-end: Parte do sistema que processa dados e lógica, geralmente oculta do usuário.
  • Front-end: Parte do sistema que interage diretamente com o usuário, como a interface gráfica.
  • QRTR: Termo específico mencionado no slide – verifique o contexto de aplicação no curso.

11. Tópicos Adicionais e Boas Práticas

Para formar uma base robusta no pensamento computacional, é importante também abordar:

  • Estruturas de Controle: Compreensão dos comandos de decisão (if, else) e repetição (loops) que direcionam o fluxo do algoritmo.
  • Comentários e Documentação: A prática de comentar o código para explicar a lógica e facilitar a manutenção.
  • Depuração (Debugging): Técnicas e ferramentas para identificar e corrigir erros no algoritmo e no código.
  • Refatoração: Melhorar a clareza e eficiência do algoritmo após a sua implementação inicial.
  • Testes: A importância de validar algoritmos com exemplos reais e casos de teste para garantir seu funcionamento correto.

Dicas para um Bom Entendimento

  • Pratique a criação de algoritmos: Tente transformar tarefas diárias em sequências de passos lógicos usando diferentes representações (linguagem natural, fluxogramas e pseudocódigo).
  • Experimente implementar algoritmos: Use linguagens de programação variadas para perceber as diferenças de sintaxe e estrutura.
  • Revise e documente: Sempre comente seu código e reescreva algoritmos para aprimorar a clareza e a eficiência.
  • Utilize recursos visuais: Diagramas, fluxogramas e pseudocódigo ajudam a visualizar a lógica e o fluxo de execução.
