Curso Módulo 2: Exemplos Práticos do Cotidiano

Descubra como algoritmos e diferentes formas de lógica estão presentes nas tarefas diárias e como esses conceitos podem ser aplicados para resolver problemas.

Introdução

Embora algoritmos sejam fundamentais para a programação, eles também regem muitas atividades cotidianas. Organizar a rotina, preparar uma refeição ou realizar tarefas simples podem ser descritas em passos lógicos. Este módulo apresenta exemplos práticos que exploram diversas formas de lógica (condicional, repetição, recursividade e funcional) e paradigmas (imperativo, estruturado, orientado a objetos e funcional).

Exemplo 1: Trocar uma Lâmpada

Descrição: Divida a tarefa de trocar uma lâmpada em passos lógicos.

  1. Verificar se a lâmpada está queimada;
  2. Desligar o interruptor;
  3. Retirar a lâmpada antiga;
  4. Instalar a nova lâmpada;
  5. Ligar o interruptor para testar.
  • Linguagem Natural: "Se a lâmpada estiver queimada, desligue o interruptor e troque a lâmpada."
  • Fluxograma:
  • Pseudocódigo: 
    Início
   Escreva "Verifique se a lâmpada está queimada."
   Se lâmpada queimada então
       Escreva "Desligue o interruptor."
       Escreva "Retire a lâmpada antiga."
       Escreva "Instale a nova lâmpada."
       Escreva "Ligue o interruptor para testar."
   Senão
       Escreva "A lâmpada está funcionando."
   FimSe
Fim

Exemplo 2: Preparar um Café

Descrição: Para fazer um bom café, cada etapa deve ser executada corretamente. Esse exemplo demonstra a aplicação de lógica sequencial.

  1. Aquecer a água até 90°C;
  2. Adicionar o pó de café no filtro;
  3. Verter a água quente sobre o café;
  4. Aguardar a infusão por 3 minutos;
  5. Servir o café.

   Escreva "Aqueça a água até 90°C."
   Escreva "Adicione o pó de café no filtro."
   Escreva "Verte a água quente sobre o café."

Exemplo 4: Verificar se um Número é Par ou Ímpar

Este exemplo utiliza lógica condicional para determinar se um número é par ou ímpar. 

Início
   Leia número
   Se (número mod 2 == 0) então
      Escreva "Número par"
   Senão
      Escreva "Número ímpar"
   FimSe
Fim

Exemplo em JavaScript:

let numero = 7;
if (numero % 2 === 0) {
  console.log("Número par");
} else {
  console.log("Número ímpar");
}

Exemplo 5: Calcular o Fatorial de um Número

O fatorial de um número (n!) é o produto de todos os inteiros positivos até esse número. Este problema pode ser resolvido utilizando recursão ou laços de repetição.

Algoritmo Recursivo em Pseudocódigo:

Função fatorial(n)
   Se n <= 1 então
      Retorne 1
   Senão
      Retorne n * fatorial(n - 1)
FimFunção

Algoritmo Iterativo em JavaScript:

function fatorial(n) {
  let resultado = 1;
  for (let i = 1; i <= n; i++) {
    resultado *= i;
  }
  return resultado;
}
console.log(fatorial(5)); // Saída: 120

Exemplo 6: Ordenação com Bubble Sort

O Bubble Sort é um algoritmo de ordenação que compara elementos adjacentes e os troca se estiverem fora de ordem. Embora simples, ilustra bem a aplicação de laços de repetição e condições.

Pseudocódigo:

Início
   Para i de 0 até n-2 faça
      Para j de 0 até n-2-i faça
         Se vetor[j] > vetor[j+1] então
            Troque vetor[j] com vetor[j+1]
         FimSe
      FimPara
   FimPara
Fim

Exemplo em JavaScript:

function bubbleSort(arr) {
  let n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    for (let j = 0; j < n - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        let temp = arr[j];
        arr[j] = arr[j + 1];
        arr[j + 1] = temp;
      }
    }
  }
  return arr;
}
console.log(bubbleSort([5, 3, 8, 4, 2])); // Saída: [2, 3, 4, 5, 8]

Exemplo 7: Paradigma Funcional – Soma de uma Lista

No paradigma funcional, o foco está em funções puras e na imutabilidade. Um exemplo clássico é a soma dos elementos de uma lista utilizando a função reduce. 

const numeros = [1, 2, 3, 4, 5];
const soma = numeros.reduce((acumulador, valorAtual) => acumulador + valorAtual, 0);
console.log(soma); // Saída: 15

Discussão: Diversas Formas de Lógica e Paradigmas

Cada exemplo apresentado neste módulo destaca diferentes formas de aplicar a lógica:

  • Lógica Condicional: Utilizada para tomadas de decisão (Ex.: verificar se um número é par ou ímpar).
  • Lógica de Repetição: Utilizada em laços (Ex.: Bubble Sort e cálculo iterativo do fatorial).
  • Lógica Recursiva: Método onde a solução chama a si mesma (Ex.: cálculo recursivo do fatorial).
  • Lógica Funcional: Baseada em funções puras sem efeitos colaterais (Ex.: soma de uma lista com reduce).

Além disso, diferentes paradigmas de programação influenciam a maneira de estruturar e resolver problemas:

  • Paradigma Imperativo e Estruturado: Foca na sequência de comandos e controle do fluxo, conforme visto em exemplos como o Bubble Sort.
  • Paradigma Orientado a Objetos: Organiza o código em objetos e classes – este paradigma pode ser explorado em módulos avançados.
  • Paradigma Funcional: Valoriza funções puras e imutabilidade, demonstrado no exemplo da soma de lista.

Compreender essas diferentes abordagens amplia sua capacidade de escolher a melhor estratégia para resolver um problema, seja em situações cotidianas ou em projetos de software.
