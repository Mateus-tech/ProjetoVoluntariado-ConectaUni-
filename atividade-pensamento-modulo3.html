Módulo 3: Problemas Lógicos

Resolva exercícios que estimulam o pensamento computacional.

Introdução

Este módulo apresenta uma série de exercícios projetados para estimular o seu pensamento computacional. Ao resolver estes problemas, você desenvolverá habilidades para identificar padrões, elaborar algoritmos e pensar de forma lógica para resolver desafios do cotidiano.

Exercício 1: Verificação de Múltiplos

Crie um pseudocódigo para verificar se um número é múltiplo de 3 e 5 simultaneamente. 

Início
   Leia número
   Se (número mod 3 == 0) E (número mod 5 == 0) então
       Escreva "O número é múltiplo de 3 e 5"
   Senão
       Escreva "O número não é múltiplo de 3 e 5"
   FimSe
Fim

Explicação: O algoritmo utiliza o operador módulo para verificar a divisibilidade por 3 e por 5. Se ambas as condições forem verdadeiras, o número é múltiplo dos dois.

Exercício 2: Relógio Digital Invertido

Imagine um relógio digital no formato HH:MM. Crie um pseudocódigo para inverter os dígitos dos minutos, mantendo a hora inalterada, e exibir a nova hora. 

Início
   Leia hora e minutos (ex.: 12 e 34)
   MinutosInvertidos ← inverter(minutos)   // Se minutos = 34, então MinutosInvertidos = 43
   Exiba hora e MinutosInvertidos (ex.: 12:43)
Fim

Explicação: O algoritmo lê a hora e os minutos, inverte os dígitos dos minutos e exibe a nova hora. A função "inverter" deve construir uma nova string com os dígitos na ordem inversa.

Exercício 3: Distribuição de Alunos

Uma turma possui 30 alunos e 5 mesas. Distribua os alunos igualmente entre as mesas e informe quantos alunos ficarão em cada mesa. 

Início
   AlunosTotais ← 30
   Mesas ← 5
   AlunosPorMesa ← AlunosTotais / Mesas
   Exiba AlunosPorMesa
Fim

Explicação: Basta dividir o número total de alunos pelo número de mesas para obter a quantidade de alunos por mesa (30/5 = 6).

Exercício 4: Verificação de Número Primo

Crie um pseudocódigo para verificar se um número é primo (um número é primo se ele só é divisível por 1 e por ele mesmo). 

Início
   Leia número
   Se número ≤ 1 então
       Escreva "Não é primo"
   Senão
       Primo ← Verdadeiro
       Para i de 2 até número-1 faça
           Se (número mod i == 0) então
               Primo ← Falso
           FimSe
       FimPara
       Se Primo for Verdadeiro então
           Escreva "É primo"
       Senão
           Escreva "Não é primo"
       FimSe
   FimSe
Fim

Explicação: O algoritmo verifica se o número é menor ou igual a 1. Em seguida, testa a divisibilidade por todos os números de 2 até (número-1). Se encontrar algum divisor, o número não é primo.

Exercício 5: A Charada dos Interruptores

Em uma sala, há três interruptores. Apenas um deles liga uma lâmpada em outra sala, e você pode visitar a outra sala apenas uma vez. Descreva um algoritmo lógico para identificar qual interruptor controla a lâmpada. 

Início
   Ligue o primeiro interruptor por alguns minutos
   Desligue o primeiro e ligue o segundo
   Vá para a sala com a lâmpada
   Se a lâmpada estiver acesa então
       O interruptor 2 a controla
   Senão, se a lâmpada estiver apagada, toque nela:
       Se estiver quente, o interruptor 1 a controla
       Se estiver fria, o interruptor 3 a controla
   FimSe
Fim

Explicação: Esse algoritmo usa a temperatura da lâmpada (quente ou fria) para determinar qual interruptor a controla, considerando o tempo de funcionamento e o fato de que somente um interruptor acende a lâmpada.

Dicas Gerais para Resolução de Problemas Lógicos

  • Divida o problema em etapas menores e resolva cada uma separadamente.
  • Utilize pseudocódigo para esboçar a solução antes de implementá-la.
  • Procure identificar padrões e relações entre os elementos do problema.
  • Teste suas soluções com diferentes exemplos para validar a lógica.

Conclusão

Resolver problemas lógicos é uma excelente maneira de desenvolver o pensamento computacional. Com prática e dedicação, você aprenderá a identificar padrões, formular algoritmos e aplicar essas habilidades para resolver desafios cada vez mais complexos.
